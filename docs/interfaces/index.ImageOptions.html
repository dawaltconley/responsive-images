Optional filenameFormat

filenameFormat?: null | ((id, src, width, format, options) => undefined | null | string)

Custom filenames

Don’t like those hash ids? Make your own!

Type declaration

    • (id, src, width, format, options): undefined | null | string

    • Parameters

      • id: string
      • src: string
      • width: number
      • format: string
      • options: Required<ImageOptions>

      Returns undefined | null | string

Param: id

hash of the original image

Param: src

original image path

Param: width

current width in px

Param: format

current file format

Param: options

set of options passed to the Image call

Example

const path = require("path");
const Image = require("@11ty/eleventy-img");

await Image("./test/bio-2017.jpg", {
  widths: [300],
  formats: [null],
  filenameFormat: function (id, src, width, format, options) {
    const extension = path.extname(src);
    const name = path.basename(src, extension);

    return `${name}-${width}w.${format}`;
  }
});

// Writes: "test/img/bio-2017-300w.jpeg"

Optional statsOnly

statsOnly?: boolean

If true, skips all image processing, just return stats. Doesn’t read files, doesn’t write files. Important to note that dryRun: true performs image processing and includes a buffer—this does not. Useful when used with urlFormat above. Better than .statsSync* functions, because this will use the in-memory cache and de-dupe requests. Those will not.

Optional svgShortCircuit

svgShortCircuit?: boolean

Skip raster formats for SVG

If using SVG output (the input format is SVG and svg is added to your formats array), we will skip all of the raster formats even if they’re in formats. This may be useful in a CMS-driven workflow when the input could be vector or raster.
